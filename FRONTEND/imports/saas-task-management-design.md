Design a modern SaaS web application for team sprint and task management similar to Jira, Linear, or Trello.

The design should include complete user flows, UI components, and developer-ready layouts.

The platform should allow users to manage projects, create teams, assign tasks, organize sprints, and collaborate.

Main features to design

Authentication
	•	Login page
	•	Sign up / registration page
	•	Password reset flow
	•	Option to link account with Telegram
	•	User profile settings

Dashboard
	•	Overview of projects
	•	Sprint progress
	•	Assigned tasks
	•	Team activity feed

Teams
	•	Create a team
	•	Invite members by email
	•	Assign roles (Admin, Developer, Viewer)
	•	Manage team members

Projects
	•	Create project
	•	Project overview page
	•	List of sprints
	•	Activity timeline

Sprint Management
	•	Create sprint
	•	Set start and end date
	•	Add tasks to sprint
	•	Sprint progress visualization

Task Management
	•	Create task
	•	Assign task to team member
	•	Set priority and status
	•	Add comments
	•	Add tags and labels
	•	Attach files

Task Board
	•	Kanban board (To Do / In Progress / Done)
	•	Drag and drop tasks

Telegram Integration
	•	Connect Telegram account via Telegram Bot
	•	Allow notifications for:
	•	Task assignment
	•	Sprint updates
	•	Mentions
	•	UI screen for managing Telegram notifications

Pages to generate
	•	Landing page
	•	Login
	•	Register
	•	Dashboard
	•	Team management
	•	Project page
	•	Sprint board
	•	Task detail page
	•	Settings

Design requirements
	•	Clean modern SaaS UI
	•	Inspired by Linear, Notion, and GitHub
	•	Dark and light mode
	•	Responsive layout
	•	Component-based design

Components to include
	•	Sidebar navigation
	•	Top navigation bar
	•	Task cards
	•	Sprint cards
	•	Team avatars
	•	Modals (create task, invite member, create sprint)
	•	Dropdowns
	•	Notification system

Developer handoff

Structure the design so it can easily translate to:
	•	React + TypeScript frontend
	•	Node.js / Express backend
	•	REST API
	•	Authentication with JWT
	•	Telegram bot integration

Include clear component structure and naming conventions for developers.
